# Bug Buddy — Marketing

## Short tagline

**Bug Buddy — Visual feedback that becomes GitHub issues.** Capture screenshots, add annotations, and turn user reports into tracked issues in one click.

---

## Paragraph (landing / about)

**Bug Buddy** is a feedback widget that turns user reports into actionable GitHub issues. Visitors click your widget, describe the problem, and get an automatic screenshot of the page—no extra steps. They can add numbered annotations to point at exactly what’s wrong. Submit once, and Bug Buddy creates a GitHub issue with title, description, screenshot, annotations, URL, and device info. Connect a GitHub App and issues stay in sync with comments and status. Add the script to your site, optionally restrict by domain and customize the widget, and start collecting structured, visual feedback where you already work: GitHub.

---

## Feature bullets

- **Screenshot + annotations** — One-click capture of the current page; users add numbered pins and notes so you see exactly what they mean.
- **Straight to GitHub** — Each submission becomes a GitHub issue (via GitHub App or OAuth) with screenshot, URL, and device/browser details.
- **Two-way sync** — Issue and comment updates in GitHub are reflected in your Bug Buddy dashboard.
- **One script, any site** — Embed a small script; optional domain allowlist and widget customization (e.g. colors, branding) keep it secure and on-brand.
- **Team-ready** — Organizations, projects, and a dashboard to triage and manage feedback before or after it becomes an issue.
